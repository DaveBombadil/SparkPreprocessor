SparkPreprocessor Configuration Examples

Common Use Cases

Basic Data Cleaning


config = {

'null_strategy': 'mean',

'outlier_strategy': 'nearest',

'standardize_text': True,

'scaling_method': 'standard'

}

Handles missing values, outliers, standardizes text fields, and scales numeric features.

High-Cardinality Categorical Handling


config = {

'max_cardinality_ratio': 0.05,

'cardinality_action': 'group',

'other_label': 'MISC',

'encode_method': 'ohe',

'explode_columns': True

}

Groups categories that make up less than 5% of data, encodes using one-hot encoding, and creates separate columns for each category.

Text-Heavy Processing


config = {

'text_avg_length_threshold': 100,

'text_max_length_threshold': 500,

'text_uniqueness_ratio': 0.9,

'text_min_length': 30,

'standardize_text': True

}

Optimized for datasets with substantial text content, identifying and processing text columns based on length and uniqueness characteristics.

High-Performance Analysis


config = {

'correlation_method': 'pearson',

'correlation_threshold': 0.8,

'scaling_method': 'standard',

'enable_polynomial_features': True,

'polynomial_degree': 2

}

Focuses on feature relationships and engineering, suitable for predictive modeling preparation.

Memory-Optimized Processing


config = {

'max_cardinality_ratio': 0.01,

'cardinality_action': 'drop',

'encode_method': 'binary',

'explode_columns': False

}

Minimizes memory usage by dropping high-cardinality columns and using binary encoding.

Strict Data Quality


config = {

'null_threshold': 0.1,

'null_strategy': 'drop',

'outlier_strategy': 'drop',

'correlation_threshold': 0.95,

'min_cardinality_ratio': 0.001,

'max_cardinality_ratio': 0.1

}

Enforces strict data quality by removing problematic data points, highly correlated features, and maintaining reasonable cardinality ranges.
